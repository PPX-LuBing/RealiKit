# Reality Scanner TUI 规划

## 一、背景

需要为 Reality 协议寻找合适的 TLS 目标域名。现有两个互补的 Go CLI 工具：

| 工具 | ⭐ | 职责 |
|------|-----|------|
| [RealiTLScanner](https://github.com/XTLS/RealiTLScanner) | 4.1k | TLS 扫描：从 IP/CIDR/域名中找出 TLS 1.3 + h2 的端点 |
| [RealityChecker](https://github.com/V2RaySSR/RealityChecker) | 660 | 深度检测：验证域名是否真正可访问、CDN 等级、被墙检测、证书有效性等 |

**现状痛点：** 两个工具都是 CLI，用户需要手动串联（先扫再验），缺少统一的交互界面。

## 二、已有相关工作

| 项目 | 语言 | 平台 | 特点 |
|------|------|------|------|
| [RealiTLScanner-GUI](https://github.com/EikeiDev/RealiTLScanner-GUI) | C# .NET 9 | Windows 仅 | 包装原 CLI，仅做 TLS 扫描，无深度检测 |
| [RealityChecker](https://github.com/V2RaySSR/RealityChecker) | Go | 跨平台 CLI | 深度检测 + CDN 等级（高/中/低），但无 TUI |

**空白地带：** 没有工具将**TLS 扫描 → 深度验证**串联成统一 TUI 界面。

## 三、方案推荐：Bubble Tea TUI

| 项目 | 说明 |
|------|------|
| 语言 | Go |
| TUI 框架 | [bubbletea](https://github.com/charmbracelet/bubbletea) + [bubbles](https://github.com/charmbracelet/bubbles) + [lipgloss](https://github.com/charmbracelet/lipgloss) |
| 扫描引擎 | 嵌入 RealiTLScanner 扫描 + RealityChecker 验证，进程内调用 |
| 跨平台 | macOS / Linux / Windows 终端 |

## 四、核心工作流：两阶段扫描

```
第一阶段：TLS 扫描（RealiTLScanner 引擎）
  ┌─ IP/CIDR ─┐   ┌─ 从文件 ─┐   ┌─ 从 URL ─┐
  └─────┬─────┘   └─────┬────┘   └─────┬────┘
        └───────────┬───┘──────────────┘
                    ▼
            TLS 1.3 + h2 扫描
                    │
                    ▼
        找到一批可行域名（CSV 输出）
                    │
                    ▼
第二阶段：深度验证（RealityChecker 引擎）
  ┌─────────────────────────────────────┐
  │ 逐个域名深度检查：                    │
  │  ✓ HTTP 可达性 & 状态码               │
  │  ✓ TLS 1.3 + X25519 + h2            │
  │  ✓ 证书有效性 & SNI 匹配              │
  │  ✓ CDN 检测（高/中/低/无）            │
  │  ✓ GFW 被墙检测                       │
  │  ✓ 地理位置（国内/国外）               │
  │  ✓ 热门网站检测                       │
  │  ✓ 重定向检测                         │
  └─────────────────────────────────────┘
                    │
                    ▼
        综合评分 + 推荐度 + 详细报告
```

## 五、综合评分体系

参考 RealityChecker 的 CDN 置信度等级，扩展为 5 星评分：

| 星级 | 含义 | 条件 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | **强烈推荐** | TLS 1.3 + h2 ✅, 非 CDN ✅, 可访问 ✅, 非热门 ✅, 国外 ✅ |
| ⭐⭐⭐⭐ | **推荐** | TLS 1.3 + h2 ✅, CDN 低置信度, 可访问 ✅ |
| ⭐⭐⭐ | **可用** | TLS 1.3 + h2 ✅, CDN 中置信度, 可访问 ✅ |
| ⭐⭐ | **勉强可用** | TLS 1.3 + h2 ✅, CDN 高置信度 或 热门网站 |
| ⭐ | **不推荐** | 缺少硬性条件 / 被墙 / 国内 / 不可访问 |

**评分因子：**
- TLS 合规（TLS 1.3 + X25519 + h2）— **硬性条件**
- HTTP 可达性（200/301/302/404 为安全状态码）
- CDN 置信度（高/中/低/无）
- 被墙状态
- 国内/国外
- 是否热门网站
- 证书有效期
- 响应延迟

## 八、交互流程

```
启动 → 输入页
  │
  ├── 选择模式：快速扫描 / 完整流程
  │
  ├── 输入目标（IP/CIDR/文件/URL）
  │
  ├── [▶ 开始扫描]
  │    ├── Phase 1: TLS 扫描（RealiTLScanner）
  │    │   ├── 实时进度 + 日志流
  │    │   └── 可行结果实时入表
  │    │
  │    ├── Phase 2: 深度验证（RealityChecker）[完整模式]
  │    │   ├── 逐个域名 HTTP 可达性检测
  │    │   ├── CDN 检测 + 被墙检测
  │    │   ├── 证书 + SNI 验证
  │    │   └── 综合评分
  │    │
  │    └── 完成 → 自动聚焦结果表格
  │
  ├── [📥 导入 CSV 验证]
  │   └── 直接对已有 CSV 执行深度验证
  │
  └── 结果管理
       ├── 按星级/域名/IP/延迟排序
       ├── 按星级/是否被墙/CDN等级过滤
       ├── 选中行 → Enter → 详情弹窗
       ├── 导出 CSV / JSON
       └── 一键复制域名列表
```

## 九、实施阶段

### Phase 1：基础框架（1-2 天）
- 搭建 Go 项目结构
- 移植 RealiTLScanner 扫描核心
- 移植 RealityChecker 验证核心
- TUI 骨架：输入视图 + 模式选择

### Phase 2：TLS 扫描（2 天）
- 实时日志展示
- 进度条
- 结果表格（可行结果实时插入）
- 扫描控制（开始/停止）

### Phase 3：深度验证（3 天）
- HTTP 可达性检测
- CDN 检测等级展示
- 被墙检测
- 证书 & SNI 验证
- 综合评分算法

### Phase 4：交互完善（2 天）
- 排序/过滤（按星级、CDN等级、是否被墙）
- 详情弹窗
- 键盘快捷键
- CSV 导入验证

### Phase 5：打磨（1-2 天）
- 错误处理
- 窗口 resize
- 测试
- README / 文档

**总计：约 9-12 天**

## 十、设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 扫描模式 | 快速(TLS only) + 完整(TLS+验证) | 用户可按需选择 |
| 验证方式 | 进程内调用 RealityChecker 库 | 避免子进程开销 |
| 评分 | 5 星制 + 多因子加权 | 直观、参考 RealityChecker CDN 等级体系 |
| 结果表 | 按星级降序排列 | 最佳结果优先展示 |
| 导入 CSV | 跳过 TLS 扫描，直接验证 | 兼容已有 RealiTLScanner 用户 |
